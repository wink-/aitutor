Tool Overview

ripgrep (rg)

A line-oriented search tool that recursively searches directories for a regex pattern. Faster than grep, ag, git grep, ucg, pt, sift.

  • Respects .gitignore
  • Unicode support
  • Parallel processing

fd

A simple, fast, and user-friendly alternative to find. Written in Rust for maximum performance.

  • Intuitive syntax
  • Colorized output
  • Smart case

fzf

A command-line fuzzy finder that can be used with any list. Interactive Unix filter for command-line.

  • Real-time preview
  • Multi-select
  • Shell integration

Installation Guide

Using Homebrew

# Install all three tools
brew install ripgrep fd fzf

# Optional: Install fzf key bindings and fuzzy completion

Ubuntu/Debian

# ripgrep
sudo apt-get install ripgrep

ripgrep (rg) - The Search Powerhouse

Basic Usage

# Basic search
rg "pattern" 

# Search in specific directory
rg "pattern" /path/to/directory

# Case-insensitive search
rg -i "pattern"

# Show line numbers
rg -n "pattern"

# Search specific file types
rg -t py "import torch"

# Exclude file types
rg -T js "TODO"

Advanced Patterns

# Regular expressions
rg "def \w+\(.*model.*\):" --type py

# Multiline search
rg -U "class.*\n.*def __init__" --type py

# Search for whole words
rg -w "train"

# Show context (3 lines before and after)
rg -C 3 "error"

# Only show filenames
rg -l "deprecated"

# Invert match (show lines NOT matching)
rg -v "test"

AI Development Use Cases

fd - The File Finder

Basic Usage

# Find files by name
fd model

# Find with extension
fd -e py

# Find in specific directory
fd model /path/to/project

# Case-insensitive search
fd -i readme

# Find directories only
fd -t d data

# Find files only
fd -t f "\.ipynb$"

Advanced Filters

# Exclude directories
fd -E "__pycache__" -E "node_modules" 

# Find files modified in last 24 hours
fd --changed-within 24h

# Find large files (> 100MB)
fd --size +100m

# Execute command on results
fd -e py -x wc -l

# Find and delete empty directories
fd -t d -e . -x rmdir

Common AI Project Searches

fzf - The Fuzzy Finder

Basic Usage

# Interactive file selection
fzf

# Preview files while selecting
fzf --preview 'cat {}'

# Select multiple files
fzf -m

# Search with initial query
fzf -q "model"

# Custom preview command
fzf --preview 'head -100 {}'

Shell Integration

# Add to .bashrc or .zshrc
# CTRL-T - Paste selected files
# CTRL-R - Search command history
# ALT-C - cd into selected directory

# Custom key bindings
export FZF_DEFAULT_OPTS='--height 40% --layout=reverse --border'

# Use fd as default source
export FZF_DEFAULT_COMMAND='fd --type f'

# Advanced preview options
export FZF_CTRL_T_OPTS="--preview 'bat --style=numbers --color=always --line-range :500 {}'"

Integration Patterns

Combining Tools

# Find Python files and search within them
fd -e py | xargs rg "neural_network"

# Interactive grep with preview
rg --files | fzf --preview 'rg --color=always -n {} {q}' --bind 'change:reload:rg --color=always -n {} {q}'

# Find large model files interactively
fd --size +10m | fzf --preview 'ls -lh {}'

# Search code and open in editor
rg -n "TODO" | fzf | awk -F: '{print $1 " +" $2}' | xargs -o vim

Custom Functions

# Add to .bashrc/.zshrc

# Interactive code search
codesearch() {
    rg --line-number --no-heading . | fzf \
        --delimiter : \
        --preview 'bat --color=always --highlight-line {2} {1}' \
        --preview-window 'up,60%,+{2}+3/3'
}

# Find and activate Python virtual environments
venv() {
    local venv_path=$(fd -t d "venv|env|virtualenv" | fzf)
    [ -n "$venv_path" ] && source "$venv_path/bin/activate"
}

# Interactive notebook opener
notebook() {
    fd -e ipynb | fzf --preview 'jupyter nbconvert --to markdown {} --stdout | head -200' | xargs jupyter notebook
}

# 1. Interactive docstring search
docstring_search() {
    rg '"""[\s\S]*?"""' --type py -U | 
    fzf --preview 'echo {} | cut -d: -f1 | xargs bat --color=always'
}

# 2. Find duplicate functions
find_duplicates() {
    rg "^def (\w+)" --type py -o -r '$1' | 
    sort | uniq -d | 
    while read func; do
        echo "=== $func ==="
        rg "^def $func\(" --type py -n
    done
}

# 3. Model checkpoint browser
checkpoint_browser() {
    fd "\.(pth|pt|ckpt|h5)$" | 
    fzf --preview 'ls -lh {} && echo && file {}'
}

# 4. Code review helper
review_changes() {
    git diff --name-only | 
    fzf -m --preview 'git diff --color=always {}'
}

# 5. Dataset explorer
explore_data() {
    fd "\.(csv|json)$" | 
    fzf --preview '
        if [[ {} =~ \.csv$ ]]; then
            head -20 {} | column -t -s,
        else
            jq . {} | head -50
        fi
    '
}

Quick Reference Cheat Sheet

ripgrep
# Basic
rg pattern

rg -w word          # whole words
rg -n pattern       # line numbers
rg -l pattern       # files only
rg -c pattern       # count matches

# Types
rg -t py pattern    # Python only
rg -T js pattern    # exclude JS

# Context
rg -A 3 pattern     # 3 lines after
rg -B 3 pattern     # 3 lines before
rg -C 3 pattern     # 3 lines both

# Advanced
rg -U pattern       # multiline
rg -v pattern       # invert match
rg --no-ignore pat  # search all
fd
# Basic
fd pattern
fd -e py           # extension
fd -t f pattern    # files only
fd -t d pattern    # dirs only
fd -H pattern      # hidden files
fd -I pattern      # no ignore

# Filters
fd -E node_modules # exclude
fd -s pattern      # case-sensitive
fd -i pattern      # case-insensitive

# Time/Size
fd --changed-within 1d
fd --changed-before 1w
fd --size +1M
fd --size -100k

# Execute
fd -x command {}
fd -X command

fzf

Best Practices for AI Development

Do's
  • Use ripgrep for searching code patterns and debugging
  • Leverage fd's gitignore support to skip irrelevant files
  • Create shell aliases for common search patterns
  • Combine tools with pipes for powerful workflows
  • Use preview features to verify results before acting
  • Take advantage of regex for complex pattern matching
Don'ts
  • Don't use grep/find when rg/fd would be faster
  • Don't ignore .gitignore patterns without reason
  • Don't search binary files or large datasets
  • Don't forget to escape special regex characters
  • Don't use overly complex patterns when simple ones work
  • Don't forget to test commands before using -x with fd
